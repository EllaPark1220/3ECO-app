## 0. 본 문서의 사용 규약

본 태스크 명세서는 SRS_V0_9의 요구사항 94건(REQ-FUNC 42 + REQ-NF 52)·UC 14건·INV 10건·API 12건·엔터티 10건을 누락 없이 분해하여 작성되었다. 각 태스크는 다음 원칙을 따른다.

1. **Contract First**: DB 스키마·API DTO·Mock은 Logic Task 보다 먼저 완료되어야 한다 (Step 1 → Step 2).
2. **Closed Context**: 동일 도메인이라도 **Read(Query) 와 Write(Command) 는 별개 태스크**로 분리되어 있다.
3. **AC = Test Code**: SRS의 AC는 단순 체크리스트가 아닌 자동화 테스트 코드 작성 태스크로 변환되어 있다.
4. **MVP 라벨 일치**: SRS_V0_9의 ✅ MVP-IN / 🟡 MVP-SOFT / ⏸️ MVP-DEFER 라벨을 태스크의 "파일럿 단계" 컬럼으로 매핑.

### 태스크 ID 체계

| Prefix | 의미 | 단계 |
|---|---|---|
| **CT-xxx** | Contract & Data (DB 스키마·API DTO·Mock) | Step 1 |
| **FR-xxx** | Functional Read (Query 전용) | Step 2 |
| **FW-xxx** | Functional Write (Command·Mutation) | Step 2 |
| **TS-xxx** | Test (TDD 시나리오·E2E·통합) | Step 3 |
| **NF-xxx** | Non-Functional (성능·보안·접근성·운영) | Step 4 |
| **IF-xxx** | Infra & DevOps (CI·Cron·환경 구축) | Step 4 |

### 복잡도 기준

- **L (Low)**: 1일 이내 단일 작업 단위. 보일러플레이트·정적 파일·환경 변수 추가
- **M (Medium)**: 2~3일. 비즈니스 로직 1개 + 기본 테스트
- **H (High)**: 4~7일. 외부 의존·다층 통합·고난도 디버깅 영역 (L2 이상)

### 파일럿 단계 매핑

|---|---|---|
