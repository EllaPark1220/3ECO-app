# 3ECO n8n 콘텐츠 자동화 파이프라인 — 전체 로드맵 v4.2

> 기준일: 2026-07-22 · 작성: Mika
> **v4.1 → v4.2 변경점**
> 1. 고성능 모델을 Claude → **GPT(OpenAI)**로 전환. Anthropic 크레딧 결제가 카드 승인 거절로 실패(해외 가맹점 FDS 추정). Anthropic Credential은 n8n에 보존 — 결제 해결 시 노드 교체만으로 복귀 가능.
> 2. 산출물 저장소를 **Google OAuth 방식**으로 확정. 서비스 계정은 자체 Drive 용량이 0이라 문서 생성이 구조적으로 불가(`storage quota exceeded`). 시트 읽기·쓰기는 서비스 계정 유지, Docs 생성만 OAuth.
> 3. 포트폴리오(GitHub 커밋) 원칙 제거 — n8n 구축에 집중. 워크플로우 JSON 백업만 유지.
> 4. Phase 0·1·2 완료 반영, 본문 분량 기준 정정(5,500 → **4,300자 ±200**).
> 일정 원칙: 노트의 구 일정(5~7월 Gantt)은 폐기. **Phase 순서 기준**으로 관리하고 날짜는 박지 않는다.

### 전 Phase 공통 원칙

- **워크플로우 JSON 백업**: 각 워크플로우 완성 시 `...` → Download로 파일 보관. 사고 대비용.
- **비밀값**: 모든 키는 secrets 폴더(레포 밖)에 파일로만 보관. 채팅·코드·시트에 붙이지 않는다.
- **인증 이원화**: 시트 읽기·쓰기 = 서비스 계정(`Google - 3eco`) / Docs 생성·수정 = OAuth(`Google OAuth - 3eco`). 노드마다 Credential을 혼동하지 않도록 주의.
- **사람 개입 지점**: 자료 픽(0-3), 게이트 1, 게이트 2. 이 셋은 자동화하지 않는다.

## Phase 2 — 자료 수집 워크플로우 ✅ 완료 (2026-07-22)

**워크플로우**: `01_자료수집` (5노드)

```
Manual Trigger
 → Google Sheets: 커리큘럼에서 episode_id 필터로 편 메타 읽기
 → OpenAI(gpt-4o): 자료 패키지 생성
 → Google Docs(OAuth): {episode_id}_자료패키지 문서 생성 + 본문 삽입
 → Google Sheets: 제작현황 '자료패키지' 컬럼 기록
```

**프롬프트 설계**: 마스터 '자료 패키지 템플릿' 준수 — ① 메타 정보(자료 비중 필수/보조 판단 포함) ② 자료 표 5~8건(자료 유형/자료명·핵심 수치/출처·발행기관/신뢰도/최신성/활용 시나리오 2~3개/시각화 가능 형태) ③ 각 자료의 학습목표 기여. 출처는 통계청·한국은행·국세청·OECD 우선, 기관명+연도 명시. 최신성 3년 이내. **불확실한 수치는 추측 금지, "확인 필요" 표시** — 환각 방지 장치이자 Ella 검증 포인트 표시.

**저장 위치**: Drive 폴더 `3ECO_자료패키지` (폴더 ID를 Expression으로 지정 — 서비스 계정 공유 폴더는 목록에 안 뜸).

**결과**: E001 자료 패키지 실산출 확인.

## Phase 3 — 책 본문 생성 워크플로우 🔄 진행 중

**선행(사람)**: 자료 픽 — 제작현황 시트 `자료선별` 칸에 Ella가 사용할 자료를 자유 텍스트로 기재. (텔레그램 다중 선택 UI는 Phase 5 이후 승격 — 아래 '승격 대기' 참조)

**워크플로우**: `02_책본문` (7노드)

```
Manual Trigger
 → Google Sheets: 커리큘럼 행 (제목·학습목표·Hook 패턴·사례 시드·연계 개념)
 → Google Sheets: 제작현황 행 (자료패키지 + 자료선별)
 → OpenAI(gpt-4o): 본문 생성
 → Google Docs(OAuth): {episode_id}_책본문 생성
 → Google Docs: 본문 삽입
 → Google Sheets: 제작현황 '책본문' 기록
```

**분량 기준 (v4.2 정정)**: **4,300자 ±200, 4단 구성**

| 단 | 분량 | 역할 |
|---|---|---|
| 도입(Hook) | 400자 | Hook 패턴별 실행 지침 적용 |
| 직관·사례 | 1,800자 | 설명 전에 장면·경험으로 감각하게 |
| 용어·개념 | 1,500자 | 여기서 처음 용어를 꺼내고 정의 |
| 마무리 | 600자 | 요약 + 일상 적용 |

> 마스터 엑셀 운영 규칙에는 "~5,500자 (5단)"으로 적혀 있으나, 채팅 합의(4,300자)와 E001·E002 실측(4,100~4,500)이 우선. **마스터 엑셀을 4,300으로 갱신할 것** — 안 하면 마스터를 다시 읽을 때마다 충돌 재발.

**Hook 실행 지침 (편별 톤 편차 방지)**
- 사례형: 구체적 한 장면에 5분 머무름. 시간·장소·행동을 적고 설명을 붙이지 않음
- 질문형: 독자가 겪었지만 말로 정리해본 적 없는 지점을 찌르는 질문 하나
- 뉴스형: 최근 현상·수치를 단서처럼 한 문장으로, 해석은 미룸
- 직설형: 개념·결론을 한 문장으로 선언하고 시작

**문체·회피 (마스터 운영 규칙 승계)**: 합니다체 통일 / 평균 문장 40자 이내 / '여러분'은 도입·전환부, '우리'는 사고 동행, 1인칭 자제 / 직관→사례→용어 순서 / 사례 1순위 청소년기~청년기, 2순위 직장·일상 / 결혼·자녀·부부 단위 회피 / 정치·종교·지역·젠더 회피 / 1권은 학자명 미사용 / 수익률·미래·이념 단정 금지 / 자료는 근거로만, 선별되지 않은 자료 미사용.

**프롬프트 보강 대기 항목** (다음 세션):
- 시각자료 마크업 형식 `> **[시각자료 N: 형식]** 설명`
- 마무리 구조 명세: 요약 → 인용블록 "이 장의 한 줄" → 적용 가이드 → 다음 편 예고 → 더 깊이 보기
- 다음 편 제목 필드 추가 (커리큘럼에서 다음 episode_id 조회 노드 1개 추가)
- 출력 형식 확정 (상단 메타 표 포함 여부 — 후단 파싱 편의)

**완료 기준**: E001 본문 초고가 Docs에 생성되고 제작현황 '책본문' 기록. 분량 4,300±200 실측 확인.

## Phase 4 — 가드레일 G1~G7 (3단 검증)

**수행 내용**: 생성된 본문을 자동 검증. 비용·성능 기준 3단 라우팅.

| 단계 | 도구 | 담당 검증 | 비용 |
|---|---|---|---|
| 1단 | 정규식 (Code 노드) | 결정론적 검출 — 회피 키워드 19종 리터럴(군 복무·군대·징병·페미·젠더·좌파·우파·진보·보수·정권·영호남·부부·자녀·아내·남편·결혼·장기 매매·마약·최저임금), 번역체 8종('매우'·'그것은 ~이다'·'~에 의해'·'결국에는'·'~을 가지고'·'~를 가지고'·'대단히'·'~에 대하여'), 금지 표현 10종 중 고정 문자열, 분량, 문장 길이(60자 초과 건수), 학자명 | 무료 |
| 2단 | 오픈소스 LLM (Llama via Groq) | 뉘앙스 판정 — 이념적 단정, 미래 단정, 과잉 친절 등 패턴이 무한한 항목의 1차 스크리닝 | 저가 |
| 3단 | GPT | 최종 품질 판단 — 학습목표 달성, 논리 구조, 2~5권 침범 여부, 2단 플래그 문장의 확정 판정 | 고성능 |

**설계 의도**: 반복 대량 스크리닝은 저비용 모델로, 판단이 어려운 최종 게이트만 고성능 모델로. 실무의 **모델 티어링/라우팅 패턴**. 채용 전형의 "오픈소스 LLM 활용" 요건 충족 근거.

**수정 루프**: 위반 발견 시 **위반 항목만** 수정하는 짧은 프롬프트로 재생성("아래 위반 항목만 수정하세요. 그 외는 건드리지 마세요"). 전체 재생성 금지 — 비용·품질 안정성. **회귀 상한 3회**, 초과 시 텔레그램으로 수동 개입 요청.

**완료 기준**: 의도적 위반 문장을 심은 테스트 본문이 각 단에서 정확히 잡히는지 단별 1건 이상 검증.

## Phase 5 — 게이트 1: 텔레그램 승인

**수행 내용**: 가드레일 통과본을 텔레그램 발송(Docs 링크 + 요약 + 승인/반려 인라인 버튼). 버튼 클릭 → n8n Webhook 콜백 → 분기. 승인 시 Phase 6, 반려 시 반려 사유와 함께 Phase 3 회귀. 결과는 제작현황 '게이트1' 기록.

**설계 원칙**: 사람 승인 없이 다음 단계로 가는 경로가 존재하지 않게 한다.

**완료 기준**: 승인·반려 양쪽 경로가 실제 텔레그램 버튼으로 1회씩 작동.

### 승격 대기 — 자료 픽 텔레그램화
게이트 1에서 텔레그램 배관(발송·인라인 버튼·Webhook 콜백)이 완성되면, Phase 3의 자료 픽을 시트 입력 → 텔레그램 선택으로 승격. **순서 이유**: 게이트 1은 2버튼(승인/반려)이라 단순, 자료 픽은 5~8건 다중 선택이라 버튼 상태 관리가 필요. 쉬운 것으로 배관을 검증한 뒤 어려운 것을 얹는다.

## Phase 6 — 병렬 제작 4트랙

**수행 내용**: 승인된 본문에서 4개 산출물 트랙을 병렬 생성. 각 트랙은 독립 서브워크플로우이며 완료 시 제작현황의 자기 상태 컬럼에 OK 기록.

- **영상 스크립트**: 본문 → 내레이션 스크립트(~1,500자 4단) + 장면 구성 데이터(Remotion 입력 JSON)
- **카드뉴스**: 카드 단위 텍스트. 1번 슬라이드 카피 25자 이내 + 인용 출처 명시, 콘텐츠 슬라이드 4장(50~80자), 6번 정체성 슬라이드. 인물 명언 회피 — 영화·드라마 명대사/고전/한국 시 중 편당 후보 3개 산출 → Ella 선택
- **OX**: 학습목표 기반 5문항 + 해설 (웹앱 FW-OX와 포맷 정렬)
- **교안/콘티**: 교안 1페이지 + 영상 콘티(1자료=1컷 룰)

**완료 기준**: E001에서 4트랙 산출물 생성 + 상태 4종 시트 기록.

## Phase 7 — Remotion 영상 렌더 (요구사항 v2)

**수행 내용**: 영상 트랙의 실제 렌더링. E001 수동 제작에서 드러난 품질 문제 해결. 착수 순서: **템플릿(1~4) → 에셋 자동화(5)**.

1. 픽토그램 물음표 버그 — 폰트·에셋 렌더 로딩 보장
2. 자막을 "지식한입" 스타일 컴팩트 박스로(하단 중앙, 밝은 배경+진한 글자, 한 줄 구어체)
3. 기존 가로형 반투명 오버레이 제거
4. 차트 3열 정렬(항목·막대·수치, 한 행=한 항목, 줄바꿈 금지)
5. 실사 에셋 확보 자동화(AI 생성 vs 스톡 API — 이 시점 결정)
6. 실사 배경 딤 처리 + 강조색 최소(수치·키워드만)
7. 상단 챕터명 표시

**Remotion의 위치**: 생성형 AI가 아니라 **렌더링 도구**. GPT가 내용·구성을 산출하고, Remotion이 그 JSON을 받아 영상으로 시공한다. 렌더 실행 위치(로컬 vs 클라우드)는 이 Phase에서 결정하고, n8n은 트리거·완료 감지만 담당.

**완료 기준**: E001 영상이 v2 요구사항 1~4 반영해 렌더 성공.

## Phase 8 — 게이트 2: 합류 감지 + 최종 검수

**수행 내용**: 스케줄 트리거로 제작현황 확인 → **상태 컬럼 4종이 모두 OK인 편**을 찾아 게이트 2 발동. 산출물 6종 링크 묶음을 텔레그램 발송, 승인/반려. 반려 시 해당 트랙만 Phase 6으로 회귀.

**완료 기준**: E001 6종 묶음 텔레그램 도착 + 승인 시 '게이트2' 기록.

## Phase 9 — 발행 3채널

- **YouTube**: Data API 업로드 (이 시점에 YouTube API 활성화 + OAuth 동의 범위 추가)
- **Instagram**: 카드뉴스 발행 (Graph API 요건 확인 후 자동/반자동 결정)
- **goyoeco.com**: 웹앱 연동 — 웹 트랙 CT-DB-003(Lesson 3매체 컬럼 additive 추가)과 합류. n8n이 완성 콘텐츠 메타데이터를 웹 DB에 반영

발행 결과는 발행_YT/발행_IG/발행_WEB + 발행일 기록.

**완료 기준**: E001이 최소 YouTube + 웹에 실발행.

## Phase 10 — 피드백 루프

일간 스케줄로 채널 지표 수집 → 운영 데이터 시트 적재. 초기엔 수집·기록까지만, 지표 기반 개선은 데이터가 쌓인 뒤 판단.

**완료 기준**: 일간 수집 3일 연속 자동 실행.
