### 1.3 §1 게이미피케이션 — 학습 흔적 모달 예외 조항 추가

PRD §1의 *"No celebrations · No 획득/점수 어휘 · Only soft fades under 300ms"* 원칙은 유지하되, **학습 흔적 모달**이라는 한정된 예외를 추가한다.

**허용 조건** (모두 충족 시 PRD §1과 정합):
- ✅ 300ms 페이드 인만 (다른 모션 — 바운스·스케일·폭죽 — 금지)
- ✅ 어휘 정합: "획득/축하/달성/완료보상" 금지 → **"흔적/마침/오늘의 한 편"** 사용
- ✅ 사용자 자유 닫기 (ESC / backdrop click / X 버튼)
- ✅ 카카오 공유는 *선택적 행동*, 자동 공유 절대 금지
- ✅ 모달 내용은 *기록형 카드* (학습 주제명·진주·요약·날짜) — *보상형 카드*가 아님

**금지 어휘 (강조)**:
- ❌ "축하합니다!" / "+10pt" / "랭킹 진입!" / "획득 완료!" / "달성하셨습니다!"
- ❌ 폭죽, 바운스, 빛 번쩍임 효과
- ❌ "오늘의 1등" / "최단 시간" / "연속 학습일" 같은 비교·자극

**근거**: 학습 완료 시 *기록*은 사용자 가치가 있고, 모달 톤만 조정하면 PRD 핵심 철학(*조용한 흔적*)과 정합 가능. 게이미피케이션 회피와 *학습 완수의 의미 부여*는 양립 가능.

### 1.4 §5 레슨 시청 — OX 통과 후 모달 명세 추가

기존 PRD §5의 *"OX 5문제 모두 정답 시 '이해 확인 완료' 메시지"* 에 다음 추가:

- in-page 메시지("이해 확인 완료")는 유지 (모달 닫은 후에도 결과 보임)
- 모달 닫기는 학습 진행을 막지 않음 (페이지에 학습 흔적 안내가 남음)

### 1.5 §6 스탬프 맵 — 진주 권별 색 적용

| 항목 | 기존 | 변경 |
|---|---|---|
| 완료 진주 색 | 청록 단일 색 | **5권 권별 다른 영롱한 색 tint** |
| 미완 진주 | 모래색 점선 outline | **권 색 점선 outline + 내부 투명** |

**권별 색 (영롱 layer tint, 베이스 흰빛 위)**:
- 1권 (돈의 언어) → 옅은 하늘
- 2권 (돈의 흐름) → 옅은 분홍
- 3권 (돈의 구조) → 옅은 노랑
- 4권 (돈의 결정) → 옅은 연두
- 5권 (돈의 인생) → 순백 (가장 영롱한 진주, 추가 색 tint 없음)

**중요한 디자인 원칙**: 진주 색은 *진주 자체의 색*이 아니라 *흰빛 진주에 비치는 영롱한 빛*이다. 즉 베이스는 모두 흰빛 광택, 그 위에 권별 색 tint가 얇게 비친다. 진주 전체를 권 색으로 칠하지 않는다.

**근거**: 권별 시각적 구분 + 진주 메타포 정확화(진짜 진주는 흰빛 위에 영롱).

## 2. SRS 변경 사항

### 2.1 학습 단위 수

- 권별 episodes 수: `[27, 25, 25, 31, 25]`

### 2.2 인증 (Auth)

- **Kakao OAuth 통합**: Kakao JavaScript SDK 사용

### 2.3 학습 흔적 모달 (Completion Modal)

**트리거**: 레슨 페이지에서 OX 5문제 모두 정답 시 (서버 검증)

**표시 데이터**:
- 권/편 메타 (예: "1권 1편")
- 학습 주제명 (편 제목 + 부제)
- 요약 2-3문장 (편별로 사전 작성된 텍스트)
- 현재 시각 (한국어 형식: `YYYY.MM.DD (요일) HH:MM`)
- 권별 색이 입혀진 조개·진주 이미지

**액션**:
- 카카오 공유 (Kakao Share SDK 사용 → 학습 흔적 이미지 카드 공유)
- 닫기 (ESC / backdrop / X 버튼 모두 동등)

### 2.4 진주 데이터 모델

- 진주 색은 *학습한 편의 권 번호*로 결정되는 클라이언트 측 lookup. 별도 DB 필드 불필요.
- 권 번호 → 색 매핑은 클라이언트 상수 (`/lib/volumeColors.ts` 같은 곳)
- 완료/미완은 `user_progress` 테이블의 `completed_at` 필드 유무로 판정

### 2.5 자산 (Assets)

- `shell_pearl.png` (480×480, ~200KB) — 학습 흔적 모달의 조개·진주 베이스 이미지
- **라이선스 확인 필요** (GPT 생성, 상업 이용 가능성 launch 전 검토) — 미해결 §4.2 참조

### 2.6 OX 퀴즈 검증

- 클라이언트 측 검증 가능하지만, **반드시 서버에서 재검증**해야 진주 부여
- 클라이언트 검증은 UX용 (즉시 피드백), 서버 검증은 무결성용

## 3. VDS 변경 사항 (VDS_v3.md → v3.1 갱신 예정)

본 항목은 별도 응답에서 `VDS_v3.md`를 직접 갱신할 예정이다. 여기서는 변경 사항만 요약.

### 3.1 진주 디자인 — 3-layer approach

```
Layer 3 (위):  작은 흰빛 highlight 반사점 (32%, 28% 위치)
Layer 2 (중):  권별 영롱한 색 tint (얇게 비침)
Layer 1 (아래): 흰빛~크림 진주 광택 베이스
              (#FFFFFF → #FBF7F0 → #ECDFCC → #C8B8A4)
```

### 3.2 권별 영롱 tint (Layer 2)

| 권 | rgba 색 | 비유 |
|---|---|---|
| 1권 | rgba(184, 218, 235, 0.32) | 옅은 하늘 |
| 2권 | rgba(240, 197, 208, 0.34) | 옅은 분홍 |
| 3권 | rgba(245, 224, 181, 0.38) | 옅은 노랑 |
| 4권 | rgba(200, 223, 196, 0.34) | 옅은 연두 |
| 5권 | (tint 없음 또는 미묘한 분홍·하늘 혼합) | 순백 진주 |

### 3.3 학습 흔적 모달 명세

- backdrop: `rgba(14, 43, 48, 0.5)` + `backdrop-filter: blur(6px)`
- card: 흰 배경, `border-radius: 22px`, 그림자 `0 24px 48px -16px rgba(13,95,109,0.3)`
- transition: `opacity 300ms ease` + `transform: translateY(12px → 0)` 동시
- 닫기 트리거: ESC / backdrop click / X 버튼 (모두 동등)
- 진주 표현: 조개 이미지 위에 권 색 layer를 `mix-blend-mode: color`로 합성

### 3.4 카카오 브랜드 색

- 카카오 노란색 `#FEE500` 유지 (브랜드 가이드 존중)
- hover: `#F5DC00`
- 카카오 버튼은 다른 화면 톤과 충돌하지 않도록 *카드 안 컴포넌트*로 격리
- 카카오 아이콘 SVG는 inline (외부 의존성 없음)

### 3.5 관리자 화면 톤

- 학습자 화면과 *시각적으로 명확히 구분*하기 위해 nav는 짙은 청록 (`--accent-darkest: #052830`) 배경
- 본문 영역은 라이트 모드 유지 (디자인 시스템 일관성)
- "Admin" 배지로 상태 명시

### 3.6 빈 상태 패턴

- 심볼: 단순 line SVG 또는 큰 타이포(예: 404) — *장식적 일러스트 금지*
- 제목: Hahmlet 600, 22~28px
- 부제: Pretendard 15px, 차분한 톤 (호들갑 금지)
- CTA: 주요 행동 1개(청록) + 보조 행동 1개(outline)
- 4종 패턴 확립: 404 / 학습 시작 전 / 로그인 필요 / 검색 결과 없음
